#  var let const

---------

## 块级作用域

**特点**
+ 不存在变量提升。
  > var会存在“变量提升”现象，即变量可以在声明之前使用，值为 undefined。
+ 暂时性死区现象。
  > 块级作用域中 let 或 const 声明的变量不受到外部影响，产生了一个封闭空间。只要变量在还没声明完成之前访问这些变量，就会报错。
  >> 本质是，只要一进入当前作用域，所使用的变量就已经存在了，但不可获取，只有等到声明变量的那行代码出现，才可以获取和使用该变量。
+ 不允许重复声明。
  > let 不允许在相同作用域内，重复声明一个变量。
+ 不绑定全局作用域
  > 在全局作用域中使用 var 声明，会创建一个新的全局变量作为全局对象的属性，而 let 和 const 不会。

## let 和 const 的区别
+ const 声明一个只读的常量，一旦声明。常量的值就不会改变。
  > const 可以修改对象的属性值，不能修改对象在栈中的引用地址。
+ const 简单类型一旦声明就不能再更改，复杂类型（数组、对象等）指针指向的地址不能更改，内部数据可以更改。

ES6 声明变量的六种方法：var、let、const、function、class、import
