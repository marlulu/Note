# 从服务器接收到请求到对应后台接收到请求
____

1. 负载均衡
   - 通过分配网络流量到多台服务器，以优化网络性能
   - 用户发起的请求都指向调度服务器，然后调度服务器根据实际的调度算法，分配不同请求给对应集群中的服务器执行，然后调度器等待服务器的HTTP响应，并将它反馈给用户
   - 形式
     - DNS负载均衡：通过DNS解析域名时但会不同的IP地址，达到用户分发到不同服务器的目的
     - HTTP重定向负载均衡：所有请求发送到一台负载均衡器，然后负载均衡将请求重定向到不同服务器上
     - 反向代理负载均衡（Nginx）：负载均衡器作为客户端和服务器之间的中间层，接收客户端的请求，并根据策略将请求转发给后端的服务器。
2. 后台的处理
   - 一般的后端有统一的验证，如安全拦截，跨域验证
   - 若这一步不符合规则，就直接返回相应的HTTP报文
   - 然后当通过验证后，才会进入实际的后台代码，然后程序接受请求，执行
   - 等程序执行完毕后，会返回一个HTTP响应包
   - 然后就是将这个包返回给前端，完成交互
